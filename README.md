# Telegram Message Monitor Bot v3.0

Продвинутый бот для мониторинга и восстановления удаленных сообщений в Telegram.

## 🎯 Возможности

### Для пользователей:
- 💾 Автоматическое сохранение всех сообщений
- 🔍 Мгновенные уведомления об удаленных сообщениях
- ✏️ Отслеживание редактирований с историей изменений
- 📸 Сохранение фото/видео с таймером
- 📊 Детальная статистика активности
- 📤 Экспорт истории чатов в TXT формате
- ⚙️ Гибкие настройки уведомлений

### Для администратора:
- 👥 Полное управление пользователями
- 📊 Глобальная статистика в реальном времени
- 📢 Массовая рассылка сообщений
- 💬 Прямая коммуникация с пользователями
- 🚫 Блокировка/разблокировка пользователей
- 📝 Логи активности
- 💾 Бэкап базы данных

## 🗄️ База данных

### Таблицы:

1. **users** - информация о пользователях
2. **messages** - все сохраненные сообщения
3. **message_edits** - история редактирований
4. **deleted_chats** - удаленные диалоги
5. **statistics** - статистика по дням
6. **user_settings** - настройки пользователей
7. **admin_notifications** - уведомления админа
8. **activity_logs** - логи активности

### Бэкап:

База данных автоматически создается при первом запуске.
Для ручного бэкапа скопируйте файл `bot_database.db`.

## ⚠️ Ограничения

1. **Telegram API:**
   - Бот не может получать события удаления в реальном времени
   - Сохраняются только сообщения, проходящие через бота

2. **Хостинг:**
   - Бесплатные хостинги могут иметь ограничения
   - Рекомендуется VPS для больших нагрузок

3. **База данных:**
   - SQLite подходит для малых/средних нагрузок
   - Для больших объемов рекомендуется PostgreSQL

## 🐛 Известные проблемы и решения

### Проблема: "TypeError: Passing parse_mode..."
**Решение:** Обновлено в v3.0.0, используйте DefaultBotProperties

### Проблема: База данных заблокирована
**Решение:** Проверьте, что запущен только один экземпляр бота

### Проблема: Бот не отвечает
**Решение:** 
1. Проверьте токен
2. Проверьте логи в bot.log
3. Перезапустите бота
